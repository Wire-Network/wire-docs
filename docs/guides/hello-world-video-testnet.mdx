# Deploy Hello World Contract on Wire Testnet

## Overview

This tutorial demonstrates how to deploy a simple `hello-world` smart contract on the Wire Testnet. We'll create a new contract account, build and deploy the contract, and then invoke its action both from the Wire Hub and via the command line using `clio`.

## Prerequisites

- Before proceeding forward, ensure that **you have completed Getting Started section** and that you have followed [Getting Started Documentation Diagram](/docs/getting-started/getting-started-intro.md).
- This page assumes you are familiar with [Smart Contract Basics](/docs/smart-contract-development/smart-contract-basics).
- A MetaMask wallet with the [MetaMask Chrome Extension](https://chromewebstore.google.com/detail/metamask/nkbihfbeogaeaoehlefnkodbefgpgknn?hl=en) installed.

:::warning[REMINDER]
The install process has already set up the wallet for the *root* user. To interact with clio, ensure **you are on the root user**. Run `sudo su -` to switch to the root user. All `clio` commands in this tutorial are prefixed with `sudo`.
:::

## Steps

### 1. Connect to the Wire Hub & Create Your Wire Testnet Account

#### 1.1. Connect Your Wallet

Navigate to [Wire Hub](https://pre.wire.network) and click the blue **Connect** button in the top right corner.

Select **MetaMask** from the wallet options and follow the prompts to:
- Add the ETH Wire Devnet network to MetaMask
- Unlock your MetaMask wallet if needed
- Confirm the signature request

<BrowserWindow url="https://pre.wire.network/">
      <img src="/img/pre-hub-wallet-connected.png" alt="pre-hub-wallet-connected" />
</BrowserWindow>

For detailed instructions, see [Connect a Wallet](/docs/wire-hub/connect-a-wallet).

#### 1.2. Create Your Wire Account

In the sidebar under **Wire Identity**, click the blue **REQUEST A WIRE USERNAME** button.

      <img src="/img/pre-hub-create-wire-username.png" alt="pre-hub-create-wire-username" />
</BrowserWindow>

Once the request is processed, you'll see a notification showing your new Wire username (e.g., "xk6j4pka6c").

### 2. Request a Developer Contract Policy

Navigate to the [Request Policy](https://pre.wire.network/network/request-policy) page from the **Contract** section in the sidebar.

<BrowserWindow url="https://pre.wire.network/network/request-policy">
      <img src="/img/pre-hub-request-developer-policy.png" alt="pre-hub-request-developer-policy" />
</BrowserWindow>

You'll see:
- **Contract Account Username**: A unique username for your contract (e.g., "contractzxne")
- **Contract Account Key Pair**: Private and public keys for your contract

**IMPORTANT**: Click **Copy Username** and **Copy Keys** to save these credentials securely. You'll need them to deploy your smart contract.

<BrowserWindow url="https://pre.wire.network/network/request-policy">
      <img src="/img/pre-hub-policy-copy-keys.png" alt="pre-hub-policy-copy-keys" />
</BrowserWindow>

Check the box *"I have saved the username and key pair to a safe place"* and click **Request New Contract Policy**.

<BrowserWindow url="https://pre.wire.network/network/request-policy">
      <img src="/img/pre-hub-policy-success.png" alt="pre-hub-policy-success" />
</BrowserWindow>

### 3. Clone the Contract Repository

Clone the existing GitHub repository containing the `hello-world` contract to your local environment:

```bash
git clone https://github.com/Wire-Network/guides.git && cd guides/hello-world-contract
```

### 4. Import the Contract Account Private Key

Ensure your wallet is unlocked:

```sh
sudo clio wallet unlock --name=default --password "$(cat /opt/wire-network/secrets/wallet_password.txt)"
```

Then, import the private key from the contract account you created in [Step 2](#2-request-a-developer-contract-policy):

```sh
sudo clio wallet import --private-key `YOUR_CONTRACT_PRIVATE_KEY`
```

### 5. Compile the Contract

Compile the contract using the build script:

```bash
./build.sh
```

Upon successful compilation, you will see a `hello/` folder with `hello.abi` and `hello.wasm` files.

```sh title="hello-world-contract/"
|   ....
├── hello
│   ├── hello.abi
│   └── hello.wasm
|   ...
```

### 6. Deploy the Contract

#### Edit the Deploy Script

Open the deploy script (`deploy.sh`) and update it to use the Wire Testnet RPC endpoint:

```sh title="deploy.sh"
sudo clio -u https://testnet-api-use1.dev.wire-dev.com set contract `CONTRACT_USERNAME` ./hello -p `CONTRACT_USERNAME`@active
```

Replace ``CONTRACT_USERNAME`` with your contract account username from Step 2.

See more information about `clio set contract` [here](/docs/api-reference/tooling/clio/command-reference/set/set-contract).

#### Run the Deploy Script

```sh
./deploy.sh
```

If successful, you'll see a transaction receipt indicating "executed transaction: `TRANSACTION_ID`".

You can view your deployed contract in the [Wire Hub](https://pre.wire.network) by navigating to **Accounts** in the explorer.

### 7. Test the Contract

#### 7.1. Test via Command Line

Push an action to invoke the `hi` action with the contract's authority:

```sh
sudo clio -u https://testnet-api-use1.dev.wire-dev.com push action `CONTRACT_USERNAME` hi '["bob"]' -p `CONTRACT_USERNAME`@active
```

Output:

```sh
executed transaction: `TRANSACTION_ID`
>> Hello, bob
```

#### 7.2. Verify in Wire Hub

Navigate to your contract's account page in [Wire Hub](https://pre.wire.network) to:
- View the transaction in the transaction history
- Inspect the contract's actions and tables
- See the console output from the action
